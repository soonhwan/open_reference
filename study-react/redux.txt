스토어, 액션, 디스패치, 리듀서, 구독
이렇게 크게 5단계의 큰 흐름으로
redux는 설계해야되고

스토어는 
상태값들을 내장하고 있고

액션은 상태값을 변화시키고 싶은놈이
호출하는 단어 입니다

디스패치는
저 액션이 스토어에 연결되는 행위를 지칭하며

리듀서는
액션이 스토어에 들어올때 실행할 일들을 담당하구요

구독은
이렇게 리듀서가 행동을 개시해서 상태가 변하면
지켜보다가 그 상태를 받아가는 요소입니다


좀 더 쉽게 설명하면
우리가 input을 통해서 onChange 라는 이벤트를 방동시키고
추가하기라는 버튼으로 list에 삽입한다 라는 이전 프로젝트에 따르면

액션(호출 하는 함수명) -> 우선 액션은 onChange가 되는거구요
스토어(state 관리) -> value 값을 state로 관리해야되니 스토어가  value 값을 가지고 있습니다
디스패치(스토어한테 액션전달) -> 액션이 일어나면 스토어한테 onchange를 디스패치 해주고
리듀서(해당액션 state 처리) -> 리듀서가 저 액션이 일어났을때 state값들을 변화시키는거죠 함수겠죠
구독(디스패치 이벤트) -> 그리고 state가 변했으니 list는 변한 state를 뿌려줘야되요 그럼 list는 구독을 하고 있는거죠 스토어를 state가 바뀌면 자기도 바꾸게
